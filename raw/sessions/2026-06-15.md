## Key facts learned (distilled into bank)
- Safety layer = two startup-only env vars out of the model's reach:
  - `MSPROJECT_SAFE_ROOT` ‚Äî confines 7 file tools (`open_project`, `save_project_as`,
    `import_xml`, `export_xml`, `export_csv`, `snapshot_to_json`, `insert_subproject`)
    via `_resolve_safe_path()`. **Unset ‚áí all file ops refused (fail-closed).**
  - `MSPROJECT_DRY_RUN` ‚Äî `_save()` skips persistence; 5 irreversible deletions
    (`delete_task`, `delete_resource`, `clear_baseline`, `delete_calendar`,
    `delete_calendar_exception`) + `close_project(save=True)` short-circuit with a preview.
  - `_launch_app()` unifies COM dispatch; `DisplayAlerts` stays off on purpose (no human
    to dismiss modal dialogs ‚Üí would hang the server).
- Residual risk: indirect prompt injection via untrusted task names/notes ‚Äî documented,
  not eliminated. Tests now require `MSPROJECT_SAFE_ROOT` + `MSPROJECT_DRY_RUN=0`.
